通过 AppCenter 集群管理控制台，您可以快速创建 MongoDB 集群。

本小节主要介绍如何快速创建 MongoDB 集群。

== 前提条件

* 已获取管理控制台登录账号和密码，且账号已实名认证。
* 已获取 MongoDB 集群操作权限。

== 操作步骤

. 登录管理控制台。
. 选择**产品与服务** > *数据库与缓存* > *文档数据库 MongoDB*，进入 MongoDB 集群管理页面。
. 点击**立即部署**，进入应用部署页面。
. 选择**区域**。
根据就近原则，选择实例所在区域。
. 配置实例基本属性、应用版本、网络信息、环境参数等信息。
+
a. <<_基本设置,基本设置>>
+
b. 节点设置，主要 <<replica-node-设置可选,Replica Node 设置>>。
+
c. <<_网络设置,网络设置>>
+
d. <<_服务环境参数设置,服务环境参数设置>>
+
e. <<_用户协议,用户协议>>

. 确认配置和费用信息无误后，点击**提交**，创建集群。
+
集群创建成功后，可在**集群管理**页面，查看和管理 MongoDB 集群。
+
image::/images/cloud_service/database/mongodb/cluster_list.png[集群列表]

=== 基本设置

集群名称、网络、版本、计费方式等基本信息配置。

[cols="1,2"]
|===
| 参数 | 参数说明

| UUID
| 系统默认分配的全局唯一标识码，不可修改。

| 名称
| （可选）输入当前集群的名称。

默认为 `MongoDB Replica`。

| 描述
| （可选）对集群的简要描述。

| 版本
| 选择集群版本。

| 快速配置
a| 选择集群资源环境类型。

可选择**测试环境**、**生产环境**两种定制资源环境。

* **测试环境**默认 1 个节点，云服务器``基础型`` 1 核 2G，硬盘``基础型本地盘`` 50G。
* **生产环境**默认 3 个节点，云服务器``企业型 e3`` 2 核 8G，硬盘``企业型 SSD 本地盘`` 200G。
* 您也可以选择 *自定义*，自定义配置资源类型。

| 计费方式
| 选择集群计费方式，可选择按**小时**、*月*、**年**计费。

合约有效期：选择按**月**、**年**计费后，即可在一个月或一年合约期内使用资源。

| 自动备份时间
| 选择集群是否自动备份。需手动选择自动备份，选择自动备份后每天指定时间段创建备份，默认自动备份为``关闭``。

| 部署方式
a| 选择集群区域部署方式，可选择``多可用区部署``和``单可用区部署``。

只有选择部署在**区域**时，才可以选择部署方式。

* 多可用区部署：将节点分散部署在当前区域的不同可用区，可用性高。 +
* 单可用区部署：将节点部署在当前区域同一个可用区，网络延迟最低。可指定节点所在当前区域的可用区

|===

=== Replica Node 设置（可选）

集群 Replica Node 节点资源配置，包括云服务器规格、磁盘大小等。

[cols="1,2"]
|===
| 参数 | 参数说明

| 云服务器类型
a| 选择集群节点云服务器资源类型，可选择**基础型**、*企业型 e3*。

include::../../../../../_components/attention_in_table.adoc[]

为保障副本的备份恢复功能，各个角色节点的**云服务器类型**配置需保持一致。

include::../../../../../_components/footer_in_table.adoc[]

| CPU
a| 选择集群节点云服务器资源 CPU 规格。

* **基础型**云服务器可选择 `1 核`、`2 核`、`4 核`、`8 核`。
* **企业型 e3** 云服务器可选择 `2 核`、`4 核`、`8 核`、`16 核`、`32 核`。

| 内存
| 选择集群节点云服务器资源内存规格。

| 磁盘类型
| 选择集群节点存储磁盘资源类型，可选择**基础型本地盘**、*企业型 SSD 本地盘*、**容量型云盘**和**通用型 SSD 云盘**。

| 磁盘大小
| 配置集群数据和日志存储磁盘大小。取值范围 30～2000GB。

磁盘大小决定了数据库最大容量以及 IOPS 能力，请根据业务量，可滑动设置或输入数字配置集群磁盘大小。

| 节点数量
a| 选择集群节点数量。 +

默认值为 3。取值范围 1、3、5、7。

include::../../../../../_components/attention_in_table.adoc[]

为保障节点高可用，建议至少配置 3 个。 1 个 节点的 MongoDB 集群仅供测试使用。

include::../../../../../_components/footer_in_table.adoc[]

=== 网络设置

通过为集群设置独享私有网络，便于网络**过滤控制**，且不影响其它私有网络的设置，可确保数据库的对不同业务进行网络隔离。数据库集群仅可加入已连接路由器的私有网络，且需确保私有网络的 DHCP 处于**打开**状态。

[cols="1,2"]
|===
| 参数 | 参数说明

| VPC 网络
a| 选择 VPC 网络。

* 默认适配同区域已有 VPC 网络。可在下拉框选择已有 VPC 网络。
* 若无可选网络，可点击**创建**，创建依赖网络资源。

| 私有网络
a| 选择私有网络。

* 默认适配VPC 网络下已有私有网络。可在下拉框选择已有私有网络。
* 若无可选网络，可点击**创建**，创建依赖网络资源。

| 节点 IP
a| 配置节点 IP 地址。

* 默认为``自动分配``。
* 选择``手动配置``需为各节点配置 IP。
|===

include::../../../../../_components/note.adoc[]

配置的**私有网络部署方式**与**集群部署方式**必须一致，即选择的集群部署方式为``多可用区部署``，则该集群仅能选择``多可用区部署``的私有网络。

include::../../../../../_components/table_footer.adoc[]

=== 服务环境参数设置

数据库的环境参数配置。

* 必须配置 *mongod 监控端口*（默认值 27017），以及数据库 *root* 和 *qc_monitor* 用户帐号密码（默认密码 `Change1Pwd`）。
* 更多参数设置和修改说明，请参见link:../../manual/config_para/config_para_info[配置参数]。
