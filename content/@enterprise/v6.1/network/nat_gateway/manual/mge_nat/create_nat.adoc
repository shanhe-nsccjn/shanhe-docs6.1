== 前提条件

已创建 VPC 网络。具体操作，请参见link:/v6.1/network/vpc/manual/vpcnet/10_create_vpc/[创建 VPC 网络]。

include::../../../../../../_components/note.adoc[]

创建 NAT 网关时，需要选择 NAT 网关所属的 VPC 网络，请在创建 NAT 网关前确保已创建好 VPC 网络。

include::../../../../../../_components/table_footer.adoc[]

== 背景信息

* 在 VPC 内创建第一个 NAT 网关时，系统会在该  VPC 的默认路由表中自动下发一条目标网络 0.0.0.0/0，下一跳指向当前创建的 NAT 网关的路由规则，用于将流量路由到 NAT 网关。因此，创建 NAT 网关后，请确保 VPC 的路由表中已存在目标网络为 0.0.0.0/0，下一跳为 NAT 网关的路由规则。如果没有该路由规则，请手动添加。具体操作，请参见link:../nat_route/[配置指向 NAT 网关的路由]。
* 如果创建 NAT 网关前，VPC 网络的路由表中已经存在目标网段为 0.0.0.0/0 的路由规则，则系统不会自动添加指向 NAT 网关的 0.0.0.0/0 路由条目。请在创建 NAT 网关后，修改原有路由条目的下一跳指向 NAT 网关。

== 操作步骤

. 登录管理控制台，在控制台导航栏中，选择**产品与服务** > *网络* > *NAT 网关*，进入 **NAT 网关**页面。
. 点击**创建**，进入**创建 NAT 网关**页面。
+
image::/images/cloud_service/network/nat/create_natgw.png[]

. 按照下表说明，配置 NAT 网关信息。
+
[cols="1,4"]
|===
| 参数 | 说明

| 区域
| NAT 网关所在的区域。

| 名称
| NAT 网关的名称。 +
支持系统自动填充及自定义。名称长度为1~64个字符，须由中文、英文字母、数字、下划线（_）、中划线（-）和点（.）组成。

| VPC   网络
| NAT 网关所属的 VPC 网络。 +
NAT 网关创建后，不支持修改 NAT 网关所属 VPC 网络。

| 规格
| NAT 网关的规格。 +
NAT 网关共有小型、中型和大型三种规格类型，不同规格的 NAT 网关会影响 SNAT 最大连接数和流量转发能力，但不会影响 DNAT 性能。请您根据业务需求灵活选择规格。规格性能数据请参考 link:/v6.1/network/nat_gateway/intro/specification/[NAT 网关规格]。

| 部署方式
a| * **多可用区部署**：NAT 网关节点分散部署在不同可用区，可用性更高。
* **单可用区部署**：NAT 网关节点部署在同一可用区，网络延迟最低。

| 可用区
| 选择**单可用区部署**方式时，才需要选择可用区。

| 公网 IP
a| NAT 网关的公网 IP。

* **购买并绑定**：购买一个新的公网 IP 并绑定到此 NAT 网关。选择后需要设置公网 IP 的计费模式和带宽上限。
* **使用已有**：若已购买有公网 IP，可选择绑定已有的公网 IP。
* **暂不绑定**：此处暂不绑定，可在创建 NAT 网关后根据需要进行绑定。具体操作请参见link:../bind_unbind_eip[绑定公网 IP]。

| 安全组
| NAT 网关所属安全组。 +
选择已有安全组。若未创建安全组，可选择默认安全组。支持创建后link:../mdfy_sg/[修改安全组]。

| 集群模式
a| 选择**多可用区部署**时，需要设置 NAT 网关节点的集群模式。

* **低延时**：网络流量会优先传输到距离最近的 NAT 网关节点上。
* **高吞吐**：网络流量会均衡地传输到所有的 NAT 网关节点上。
|===

. 点击**立即创建**。
+
待创建成功后，可以在 NAT 网关列表中看到已创建好的 NAT 网关。
