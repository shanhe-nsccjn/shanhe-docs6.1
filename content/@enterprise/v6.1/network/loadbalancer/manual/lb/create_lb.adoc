负载均衡器实例是运行负载均衡服务的实体，搭建负载均衡器服务您需要先创建负载均衡器实例。

== 前提条件

已创建私有网络或公网 IP。具体操作请参见link:/v6.1/network/vpc/manual/vpcnet/10_create_vpc/[创建 VPC 网络]及link:/v6.1/network/eip/manual/ipv4/outband_ipv4/#申请外部绑定的-ipv4-公网-ip[创建公网 IP]。

include::../../../../../../_components/note.adoc[]

创建公网负载均衡器需要绑定公网 IP，创建私有网络负载均衡器需要绑定私有网络，创建公网+私网负载均衡器则需要同时绑定公网 IP 及私有网络。

include::../../../../../../_components/table_footer.adoc[]

== 操作步骤

. 登录管理控制台，选择**产品与服务** > *网络* > *负载均衡器*，进入**负载均衡器**页面。
. 点击**创建**，弹出**创建负载均衡器**页面。
+
image::/images/cloud_service/network/lb/create_lb.png[,50%]

. 设置负载均衡器参数。
+
参数说明如下表。
+
[cols="1,3"]
|===
| 参数 | 说明

| 名称
| 负载均衡器实例名称。

| 节点最大连接数
a| 单个负载均衡节点能够承载的最大连接数量。 +
当实例上的连接超过规格定义的最大连接数时，新建连接请求将被丢弃。

include::../../../../../../_components/note_in_table.adoc[]
此处的并发规格适用于非 HTTPS 场景的监听器。 +
对于 HTTPS 监听器，由于需要在负载均衡器上做证书卸载，耗费更多的 CPU，因此其并发规格为负载均衡器规格的一半。

include::../../../../../../_components/footer_in_table.adoc[]

| 节点数量
| 每个公网 IP 下节点数量，节点数大于 1 时，负载均衡器以多活方式部署。推荐设置为大于 1。 +
总节点数 = 公网 IP 数量 * 节点数量 +
并发能力 = 节点最大连接数 * 总节点数

| 部署方式
a| 节点部署方式。

* 多可用区部署：节点分散部署在不同可用区。可用性高。
* 单可用区部署：节点部署在一个可用区中，网络延迟最低。

| 可用区
| 选择单可用区部署时，需要选择具体的可用区。

| 公网 IP
| 负载均衡器绑定公网 IP 后可以接收来自公网的访问请求并自动分发到多台后端服务器。

| 网络
| 负载均衡器所属私有网络。若不加入私有网络，则默认使用基础网络。私有网络适合网络自定义配置的场景。

| 安全组
| 负载均衡器所属安全组。

| *高级选项*
| 点击**显示高级选项**，进行以下高级选项配置。

| 后端网络类型
| 负载均衡器实例的 IP 版本。 +
可选择 IPv4 或 IPv6 网络，默认使用 IPv4 网络。

| WAF
| 选择是否开启 Web 应用防火墙（WAF）能力，若开启，需要选择已创建好的 WAF 配置组。

| HTTP 头部限长
| 当负载均衡器代理 HTTP/HTTPS 协议时，头部限制的最大长度。 +
系统默认值随版本而定，一般是 15 KB；选择自定义时，自定义范围为 1-127 KB，调高此参数时会影响最大连接数， 建议使用默认值。
|===

. 点击**提交**，开始进行创建并返回负载均衡器页面。
