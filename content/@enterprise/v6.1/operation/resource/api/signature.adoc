这里介绍 API 请求中签名（signature） 的生成方法。您可以根据实际情况选择使用 **API 密钥**或 link:/v6.1/authorization/iam/[IAM] **身份**来完成签名。

== API 密钥签名

您需要先在控制台创建 API 密钥，获得 ``accesss_key_id`` 和 ``secret_access_key``，这里假设：

[source]
----
access_key_id = 'QYACCESSKEYIDEXAMPLE'
secret_access_key = 'SECRETACCESSKEY'
----

请求参数如下:

[source]
----
{
  "count":1,
  "vxnets.1":"vxnet-0",
  "zone":"jn1a",
  "instance_type":"small_b",
  "signature_version":1,
  "signature_method":"HmacSHA256",
  "instance_name":"demo",
  "image_id":"centos64x86a",
  "login_mode":"passwd",
  "login_passwd":"login20130712",
  "version":1,
  "access_key_id":"QYACCESSKEYIDEXAMPLE",
  "action":"RunInstances",
  "time_stamp":"2013-08-27T14:30:10Z"
}
----

include::../../../../../_components/note.adoc[]
你可以使用上述的 ``AccessKey`` 和 ``Request`` 调试你的代码，当得到跟后面一致的签名结果后（即表示你的代码是正确的），可再换为你自己的 ``AccessKey`` 和其他 API 请求。

include::../../../../../_components/table_footer.adoc[]


=== 步骤 1：按参数名进行升序排列

排序后的参数为:

[source]
----
{
  "access_key_id":"QYACCESSKEYIDEXAMPLE",
  "action":"RunInstances",
  "count":1,
  "image_id":"centos64x86a",
  "instance_name":"demo",
  "instance_type":"small_b",
  "login_mode":"passwd",
  "login_passwd":"login20130712",
  "signature_method":"HmacSHA256",
  "signature_version":1,
  "time_stamp":"2013-08-27T14:30:10Z",
  "version":1,
  "vxnets.1":"vxnet-0",
  "zone":"jn1a"
}
----

=== 步骤 2：参数 URL 编码

对参数名称和参数值进行 URL 编码，编码后的请求串为:

[source]
----
{
  "access_key_id":"QYACCESSKEYIDEXAMPLE",
  "action":"RunInstances",
  "count":1,
  "image_id":"centos64x86a",
  "instance_name":"demo",
  "instance_type":"small_b",
  "login_mode":"passwd",
  "login_passwd":"login20130712",
  "signature_method":"HmacSHA256",
  "signature_version":1,
  "time_stamp":"2013-08-27T14%3A30%3A10Z",
  "version":1,
  "vxnets.1":"vxnet-0",
  "zone":"jn1a"
}
----

include::../../../../../_components/note.adoc[]
编码时，空格要转换成 ``%20``，而不是 ``+``。

include::../../../../../_components/table_footer.adoc[]

include::../../../../../_components/attention.adoc[]
转码部分的字符要用大写，如 ``:`` 应转成 ``%3A``，而不是 ``%3a``。

include::../../../../../_components/table_footer.adoc[]

=== 步骤 3：构造 URL 请求

参数名和参数值之间用 ``=`` 号连接，参数和参数之间用 ``＆`` 号连接，构造后的 URL 请求为

[source]
----
access_key_id=QYACCESSKEYIDEXAMPLE&action=RunInstances&count=1&image_id=centos64x86a&instance_name=demo&instance_type=small_b&login_mode=passwd&login_passwd=login20130712&signature_method=HmacSHA256&signature_version=1&time_stamp=2013-08-27T14%3A30%3A10Z&version=1&vxnets.1=vxnet-0&zone=jn1a
----

=== 步骤 4：构造被签名串

被签名串的构造规则为: 被签名串 = ``HTTP请求方式`` + ``\n`` + ``uri`` + ``\n`` + ``url 请求串``

include::../../../../../_components/attention.adoc[]
"`\n`" 是换行符，不要将 "`\`" 转义。也就是说，不要用 "`\n`" ，有些语言，比如 php 和 ruby，请用 "`\n`" , 而不是 '`\n`'。

include::../../../../../_components/table_footer.adoc[]

假设 HTTP 请求方法为 ``GET`` 请求的 uri 路径为 "`/iaas/`", 则被签名串为

[source]
----
GET\n/iaas/\naccess_key_id=QYACCESSKEYIDEXAMPLE&action=RunInstances&count=1&image_id=centos64x86a&instance_name=demo&instance_type=small_b&login_mode=passwd&login_passwd=login20130712&signature_method=HmacSHA256&signature_version=1&time_stamp=2013-08-27T14%3A30%3A10Z&version=1&vxnets.1=vxnet-0&zone=jn1a
----

=== 步骤 5：计算签名

计算被签名串的签名 signature。

* 将 API 密钥的私钥（secret_access_key） 作为 key，生成被签名串的 HMAC-SHA256 或者 HMAC-SHA1 签名，更多信息可参见 http://www.ietf.org/rfc/rfc2104.txt[RFC2104]。
* 将签名进行 Base64 编码。
* 将 Base64 编码后的结果进行 URL 编码。

include::../../../../../_components/attention.adoc[]
当 Base64 编码后存在空格时，不要对空格进行 URL 编码，而要直接将空格转为 ``+``。

include::../../../../../_components/table_footer.adoc[]

以 Python（版本 2.7）代码为例（其他语言类似，需要使用 ``sha256`` + ``base64`` 编码，最后需要再进行  编码，URL 编码时需要将原有的空格转为 ``+``）。

[source]
----
import base64
import hmac
import urllib
from hashlib import sha256

# 前面生成的被签名串
string_to_sign = 'GET\n/iaas/\naccess_key_id=QYACCESSKEYIDEXAMPLE&action=RunInstances&count=1&image_id=centos64x86a&instance_name=demo&instance_type=small_b&login_mode=passwd&login_passwd=login20130712&signature_method=HmacSHA256&signature_version=1&time_stamp=2013-08-27T14%3A30%3A10Z&version=1&vxnets.1=vxnet-0&zone=jn1a'
h = hmac.new(secret_access_key, digestmod=sha256)
h.update(string_to_sign)
sign = base64.b64encode(h.digest()).strip()
signature = urllib.quote_plus(sign)
----

=== 步骤 6：添加签名

将签名参数附在原有请求串的最后面，最终的 HTTP 请求串为（为了查看方便，我们人为地将参数之间用``回车键``分隔开）

[source]
----

&action=RunInstances
&count=1
&image_id=centos64x86a
&instance_name=demo
&instance_type=small_b
&login_mode=passwd
&login_passwd=login20130712
&signature_method=HmacSHA256
&signature_version=1
&time_stamp=2013-08-27T14%3A30%3A10Z
&version=1
&vxnets.1=vxnet-0
&zone=jn1a
&signature=byjccvWIvAftaq%2BoublemagH3bYAlDWxxLFAzAsyslw%3D
----

&instance_name=demo

&login_mode=passwd
&login_passwd=login20130712
&signature_method=HmacSHA256
&signature_version=1

&version=1

这里我们假设您的 API 执行设备为 id 为 ``i-7lchv5u3`` 的云服务器。
